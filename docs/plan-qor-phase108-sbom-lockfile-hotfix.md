## Context

Phase 107's `requirements-sbom.txt` was generated by `pip-compile --generate-hashes` on Python 3.13. The compiler did not lock the `typing_extensions<5.0,>=4.6` transitive dependency of `cyclonedx-python-lib[validation]==11.7.0` because Python 3.13 stdlib already provides the relevant typing features, so the resolver treated the bound as satisfied at compile time.

The release workflow runs on Python 3.12 (per `.github/workflows/release.yml`). On 3.12 the resolver still pulls `typing_extensions` as a runtime dep of `cyclonedx-python-lib[validation]`. The `pip install --require-hashes -r requirements-sbom.txt` step in the build job exits non-zero with:

> ERROR: In --require-hashes mode, all requirements must have their versions pinned with ==. These do not: typing_extensions<5.0,>=4.6 ...

This blocks the v0.75.0 PyPI publish (build job FAILED -> publish job SKIPPED). The v0.75.0 git tag is live; PyPI does not have v0.75.0. Pattern matches the Phase 104 publish-step recovery (botched publish requires a hotfix bump because PyPI versions are immutable once published, and `setup.py sdist` runs already produced a v0.75.0 sdist name claim even though it never uploaded).
