## Executive Summary

`/qor-substantiate` Step 9.5's documented `git add` block enumerates seven paths
and never stages the sealed session's `.qor/gates/<sid>/` directory -- yet Step
7.8 and the CI completeness job make exactly those files load-bearing for every
future run on the sealed branch. Recent seals include the gate dirs only because
operators stage them OUTSIDE the documented procedure; a consumer following the
prose to the letter ships a seal whose CI completeness gate fails. The fix is a
one-block prose amendment under a hard constraint: the file has 31 bytes of
headroom before the 40 KB EXCEEDED budget, so the amendment must compact the
existing block to pay for itself.

## Findings

### F1. The omission (verified live)

- `qor/skills/governance/qor-substantiate/SKILL.md:589-600`: the Step 9.5 block
  lists exactly `CHANGELOG.md`, `docs/CONCEPT.md`, `docs/ARCHITECTURE_PLAN.md`,
  `docs/META_LEDGER.md`, `docs/SYSTEM_STATE.md`, `docs/BACKLOG.md`, `src/`.
  No `.qor/gates/` reference anywhere in the step.
- Load-bearing consumer: `qor/reliability/gate_chain_completeness.py:52` walks
  every SESSION SEAL >= phase 52 and requires the session's gate artifacts in
  the COMMITTED tree; `.github/workflows/ci.yml` runs it as a required job. The
  local Step 7.8 check passes on disk contents, so the omission is invisible
  until CI rejects the pushed seal (the incident shape GH #262 records).
- Recent seal commits (e.g. the Phase 173-175 checkpoints) DO contain the gate
  dirs -- staged manually outside the documented list, which is precisely the
  hidden operator burden the issue names.

### F2. The byte-budget constraint (verified live)

- Current size: 40,929 bytes; EXCEEDED threshold 40,960
  (`qor/scripts/skill_size_budget_lint.py:24`): 31 bytes of headroom.
- The existing seven single-path `git add` lines total ~188 bytes; consolidating
  to two multi-path lines (~165 bytes incl. the new `".qor/gates/$SESSION_ID/"`
  argument) pays for the addition; a short `(uses $SESSION_ID from Step 4.6)`
  pointer keeps the net delta ~+12 bytes -- under EXCEEDED. The deeper trim is
  GH #266's scope (queue-later), not this fix's.
- `$SESSION_ID` is already resolved earlier in the same protocol run
  (`SKILL.md` Step 4.6 canonical helper line), so the block can reference it.

### F3. No test locks the staging list

- `tests/test_substantiate_changelog_integration.py:65-106` mirrors the
  auto-stage of CHANGELOG.md only; `tests/test_seal_flow_ordering.py` locks
  step ordering tokens. Nothing asserts `.qor/gates/` membership in Step 9.5 --
  the regression class is unguarded. New wiring test must carry a
  `# prose-lint: ok=...` reason (prose-contract test class, 54 precedents).

### F4. Variant propagation

- `qor/dist/variants/{claude,codex,kilo-code}/skills/qor-substantiate/SKILL.md`
  are identity copies regenerated by `qor-logic scripts dist_compile` (seal Step
  8.5) -- the canonical edit propagates automatically; `check_variant_drift`
  guards equality.
