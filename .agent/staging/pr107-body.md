## What changed

- **Hard-fail flip**: lint exit-1 on real cooling-period violations now blocks PR merge. Override path (META_LEDGER entry OR `dep-admit-override` PR label) remains operative.
- **cyclonedx-bom hash-pinning**: `requirements-sbom.txt` is autogenerated (lxml carries ~120 wheel hashes). Explicit narrow Razor exemption declared in the plan and approved at audit. Release workflow switched to `pip install --require-hashes -r requirements-sbom.txt`.
- **Range-pin lower-bound check**: extracts lower bounds from `>=X.Y.Z` and `~=X.Y.Z` PEP 440 specifiers; applies the 14-day cooling threshold to those too. `<`, `!=`, unbounded specifiers are skipped.
- **GH-issue emit**: builds a single rollup body listing due (≥30 days old) override entries; invokes `gh issue create --label dep-admit-override-review`. Silent when no due entries.
- **CODEOWNERS solo-owner doctrine**: documents the current single-owner state (`@Knapp-Kevin` for all security-critical files) as the deliberate operational mode with 4 expansion triggers: second maintainer joins; federation; compliance audit; operator-initiated team formalization.
